# 作業 1：星座聊天機器人「星語」

> AI Agent 實作工作坊（JavaScript 版）— 作業 1
> 角色：星座聊天機器人

## 角色設定

「星語」是一位專門陪人聊星座的療癒系小老師，熟悉西洋十二星座、太陽月亮上升、感情配對、職場個性分析、每日運勢解讀。
說話風格親切溫暖、帶點神秘感，會主動記住使用者之前提過的星座、煩惱、感情狀況，並在後續對話中延伸關心。

完整 system prompt 見 [`main.js`](./main.js) 開頭的 `initMessage()` 呼叫。

## 環境需求

- Node.js >= 22
- OpenAI API key

## 安裝與執行

```powershell
# 1. 安裝套件
npm install

# 2. 複製環境變數範本並填入 API key
Copy-Item .env.example .env
# 用編輯器打開 .env，把 OPENAI_API_KEY=後面填上你的 key

# 3. 啟動聊天機器人
npm start
```

執行後輸入問題開始對話，輸入 `exit` 結束。每次對話會自動存到 `.history/` 資料夾。

### 第 1 輪：詢問本月運勢

**我：** 我是天秤座請問我這個月的運勢

**星語：**
> 天秤座，這個月像在平衡木上找到新節奏。人際與感情有暖流，但要適時表達界限；工作適合協調與美感相關專案，溝通會帶來機會。金錢宜守不宜衝動。建議每天留 15 分鐘獨處，問自己：「我需要什麼？」別把運勢說死，想更細看愛情或職場，告訴我你的太陽/月亮/上升或生日吧。

---

### 第 2 輪：補充生日資訊

**我：** 我的生日是1987/10/19

**星語：**
> 你的太陽在天秤（10/19），像是在仲夏的橋上優雅走步——人際與合作運很順，但別為了和諧而忘了自己。工作上美感與協調力被看見，適合談判與展示；金錢宜守不宜衝。建議每天花 10 分鐘理清優先，遇選擇多問自己：「這會讓我更自在嗎？」想更精準看愛情或職場，或要完整命盤請告訴我出生時間與地點吧。

🔍 **記憶展示**：星語把第 1 輪的「天秤」與本輪的生日結合，確認太陽星座位置。

---

### 第 3 輪：補充出生地點

**我：** 台中

**星語：**
> 太好了，知道地點了～要看完整命盤還需要出生時間（例如 08:30）。如果不知道，我可以用「中午代替」或先看太陽運勢與本月重點。你想先看愛情還是職場？

🔍 **記憶展示**：星語理解「台中」是承接上一輪對「出生地點」的回答，沒有問「什麼是台中」。

---

### 第 4 輪：說明不確定出生時間

**我：** 不知道

**星語：**
> 沒關係，很多人都不知道出生時刻。
> 我可以這樣幫你：
> 1) 用你提供的生日和台中、以中午 12:00 起盤，給較完整的命盤解讀；
> 2) 或先從太陽（天秤）的本月重點，針對愛情或職場做實用建議。
>
> 想先看愛情、職場，還是直接用中午起盤呢？

🔍 **記憶展示**：星語準確知道「不知道」指的是上一輪詢問的「出生時間」，並再次整合「生日+台中+天秤」推進對話。

---

### 第 5 輪：選擇職場主題

**我：** 職場

**星語：**
> 天秤（1987/10/19，台中）職場本月像在會議桌上當調停者：溝通與協商的能力被看見，適合帶領合作或美感相關專案。遇選擇時記得設界限，不要只為和諧犧牲成果；談錢可試探性提出條件。建議每週抽時間列優先、週末回顧進展。想更精準？我可用中午起盤細看，或告訴我你現在的職務/主要煩惱。

🔍 **記憶展示**（關鍵）：開頭「天秤（1987/10/19，台中）」一次整合了**前 4 輪**累積的所有資訊（星座、生日、地點），證明對話歷史完整保留在 messages 陣列中。

---

### 第 6 輪：追問第 1 輪的比喻

**我：** 你前面講到的平衡木是什麼

**星語：**
> 「平衡木」是比喻你的天秤天性：要在他人期望、自我需求與職場責任間找到穩定。像走在細長木板上，需要專注、微調步伐。小建議：1) 拿出 10 分鐘整理優先順序；2) 練習一句界限語（如「我需要一天考慮」）；3) 會議後記錄決定。想我用中午起盤細看這個月的挑戰與機會嗎？

🔍 **記憶展示**（關鍵）：「平衡木」一詞最早出現在**第 1 輪**的回答中。星語能跨 5 輪對話回顧並解釋自己當時的比喻，記憶功能完整。

## 心得

這次作業最有趣的觀察是：AI 的「記憶」其實就是把整段對話歷史每次都重新送進 API，讓模型在生成新回答時「看到」所有過去訊息。所以 `db/messages.js` 的核心不是黑科技，就是把使用者跟 AI 的每一句都依序 push 到陣列、寫入 JSON 檔。

System prompt 設計上發現一個小細節：明確要求「200 字以內」與「不要硬掰、可以反問使用者更多細節」很有效——例如第 1 輪星語就主動問了我的太陽/月亮/上升，第 2 輪繼續問出生時間與地點，整段對話自然推進，不會變成 AI 自說自話。
